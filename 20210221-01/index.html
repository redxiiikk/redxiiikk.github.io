Github Actions 与 Checkstyle

团队合作是项目开发过程中不可避免的事情，因此一个统一的代码风格就显得格外的重要，可以有效的避免团队中所有的成员写出来的代码风格各不相同，减少合作之间的摩擦。

在 Java 的世界中，Checkstyle 是最常使用的代码风格检查插件，因此本文将使用 Github Actions 在提交代码的时候进行 Checkstyle 检查。

Github Actions

Github Actions 是 Github 推出的持续集成服务，具体的资料可以查看官网文档，本文将直接开始使用 Github Actions。

CLI 运行

通常会在 IDE 中使用插件去检查 Checkstyle，但是在 Github Actions 中无法使用该种方式，必须使用 CLI 模式去运行 Checkstyle，具体的方式如下：

  1. 下载 checkstyle 的 jar
    checkstyle 的代码托管在 Github 上，这样就可以从 Github Releases 中直接下载最新的 jar。

    Releases · checkstyle/checkstyle

  2. cli 模式下进行 checkstyle 检查

    CLI 中进行 checkstyle 检查也十分的简单，官方文档中也对这种方式进行了说明：

reviewdog

通过在 Github Actions 中直接以 CLI 的方式运行 Checkstyle 已经可以对代码的风格进行检查了，但是这样的检查结果将直接显示在 Gihtub Actions 日志中，不利于查看，这是就要使用 reviewdog 将检查的结果直接显示在 Gihtub Actions 界面中：

同时也会在提交记录中显示运行状态：

reviewdog 已经适配了 Checkstyle 的输出格式，因此可以直接使用。

总结

当然上述方法仍有不足之处，依然需要不断的改进：

  1. 代码风格检查未通过时应该不允许提交代码

  2. 检查结果通知提交人
